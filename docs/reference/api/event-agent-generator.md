# API Reference: @receptron/event_agent_generator

This page outlines the conceptual API for the `@receptron/event_agent_generator` package, which provides Vue-based tools for generating GraphAI agent configurations.

## Overview

`@receptron/event_agent_generator` is designed to simplify the creation of agent JSON structures for GraphAI workflows, likely through a user interface or a set of programmatic utilities. It seems to have a particular focus on event-related agents and text input agents.

Given its Vue.js nature and file structure (`App.vue`, `main.ts`, `src/event_agent_generator.ts`, `src/text_input_agent_generator.ts`), it could be:
1.  A standalone Vue application that users run to generate configurations.
2.  A library of Vue components or composables that can be embedded into other Vue applications.
3.  A set of JavaScript/TypeScript classes or functions for programmatic generation.

The compiled output in `lib/` suggests it exports modules.

## Conceptual API Components

### 1. Vue Components (If Exported for Embedding)

#### `<AgentGeneratorUI />` (Conceptual Top-Level Component)
A primary Vue component that encapsulates the entire agent generation interface.
-   **Props**:
    -   `agentSchemas?: Record<string, AgentSchema>`: An object where keys are agent type IDs and values are schemas defining their parameters, inputs, etc. This would allow customization of available agents.
    -   `initialConfig?: PartialAgentConfig`: Pre-fill the generator with an existing partial configuration.
-   **Events**:
    -   `@generated(config: FullAgentConfig)`: Emitted when a complete agent configuration is generated by the user.
    -   `@updated(config: PartialAgentConfig)`: Emitted as the user makes changes in the UI.

#### `<AgentTypeSelector :schemas="agentSchemas" @selected="handleTypeSelect" />` (Conceptual)
A component to select the type of agent to configure.

#### `<DynamicForm :schema="currentAgentSchema" v-model="agentParams" />` (Conceptual)
A component that renders a form based on the selected agent's parameter schema.

## Usage Scenarios

**Scenario A: Using a Standalone UI Tool**
-   Developer runs the Event Agent Generator application (e.g., `npm run dev` in its package directory).
-   Interacts with the web UI:
    1.  Selects an agent type (e.g., "Webhook Listener", "User Input Form").
    2.  Fills out a dynamically generated form for the agent's parameters (e.g., webhook URL, form field definitions).
    3.  The UI displays the resulting JSON for the GraphAI agent node.
    4.  Developer copies this JSON and pastes it into their `GraphAIConfig` file.
